# Raspberry Pi(Linux) 전용 하드웨어 의존성.
# macOS/CI에서는 설치하지 않는다(Mock 구현으로 동작). pyproject가 아닌 별도 파일로 관리하는 이유:
# 이 패키지들의 sdist는 Linux에서만 빌드되어 uv universal 해석을 깨뜨리기 때문.
#
# Pi에서 설치(venv = 시스템 Python 3.13 + --system-site-packages, docs/pi-setup.md §10):
#   uv venv --python /usr/bin/python3 --system-site-packages
#   uv sync --inexact                      # 크로스플랫폼 + dev 도구 (system-site 보존됨)
#   uv pip install -r requirements-pi.txt  # 아래 하드웨어 의존성
#
# ⚠️ bare `uv sync`는 락파일에 없는 이 패키지들을 prune(제거)해 런타임을 깬다.
#    반드시 `uv sync --inexact`를 쓰거나, uv sync 후 이 파일을 재설치할 것.
#
# picamera2 / libcamera 는 여기에 두지 않는다 — apt(python3-picamera2, python3-libcamera)로
# 설치하고 --system-site-packages 로 import 한다. 이유: libcamera Python 바인딩은 PyPI에 없고,
# 시스템 libcamera에 대해 빌드돼야 하는데(rpi-libcamera) Debian의 libcamera-dev가 내부 헤더
# (base/log.h 등)를 누락해 venv 빌드가 불가하다. apt 패키지는 전체 소스 빌드 산물이라 문제없다.
# (그래서 venv도 apt와 같은 시스템 Python 3.13을 쓴다 — pyenv 3.12.10은 ABI 불일치)
